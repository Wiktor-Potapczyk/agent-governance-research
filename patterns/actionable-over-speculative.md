# Actionable Fixes Over Speculative Architecture

Interesting architectural possibilities should not drive the work queue. Concrete gaps in the current system produce immediate value when fixed. Speculative redesigns produce specs that can't be tested.

## The Principle

When a conversation drifts into future-model design, save it and redirect: "What actionable fix can we do with the current setup?"

## Priority Order

1. Fills gaps (missing data, missing checks)
2. Ensures correctness (existing mechanisms work right)
3. Adds capability (new mechanisms)
4. Redesigns architecture (only when 1-3 are exhausted)

## How to Apply

Speculative ideas (multi-agent coordination, collaborative decomposition, terminal synthesizers) are worth saving as design notes. They should not be prioritized over concrete gaps like missing event logging, incomplete governance data, or unrecorded quality metrics. Fix what exists before designing what might exist.
