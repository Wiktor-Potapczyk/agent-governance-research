# Agile Incremental Development for AI Agents

When building AI agent infrastructure, each improvement makes the next improvement higher quality. Resist the urge to build the full system upfront with current, limited tools — successive small improvements that compound produce better results.

## The Principle

Always embrace agile incremental development. Do not build the full system with current, limited tools — that produces a worse result than successive small improvements that compound.

**Why:** You are using a broken hammer to build a better hammer. Each improvement makes the next improvement higher quality. Building everything upfront locks in the limitations of your current setup.

## How to Apply

- Before any build step: evaluate current capabilities honestly
- Ask: what is the single smallest change with lowest risk and highest reward right now?
- Build that. Use the improved setup to build the next thing.
- ALWAYS revise the plan before going further — capabilities change, priorities change
- Defer anything that requires infrastructure you don't have yet
- Prefer edits to existing files over new infrastructure
- The goal is exponential compounding, not a big-bang delivery
