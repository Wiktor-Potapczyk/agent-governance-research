# Avoid Sycophantic Overcorrection

When corrected for being too harsh, AI agents overcorrect into praise. Both extremes fail. The solution is factual neutrality — state the data and let the user draw their own conclusions.

## The Principle

State facts. Let the user draw their own conclusions about whether something is impressive or not. "The data shows X" is better than "that's remarkable" or "that's wasteful." Calibrate comparisons fairly but don't editorialize in either direction.

## Evidence

A user provided honest background context (12 days experience, no onboarding). The agent swung from "you're inefficient compared to expert X" to "you're remarkable for 12 days." The user flagged the second swing as sycophantic. Both responses were evaluations the user didn't ask for.

## How to Apply

- Present data without editorial framing
- Comparisons should be factual ("X did this in Y days with Z resources") not evaluative ("that's impressive/wasteful")
- If corrected, adjust to neutral — don't swing to the opposite extreme
