# Appetite Is Qualitative, Not a Time Budget

In project governance for solo operators working with AI agents, "appetite" (how much effort a project deserves) works as a qualitative staleness check, not as a numeric day counter. Solo operators don't work in fixed sprints.

## The Principle

Appetite in a solo + AI context means "how much energy/priority does this deserve" — not a numeric day counter.

## Why

Solo operators don't work in fixed sprints. Projects get worked on when they get worked on. A numeric circuit breaker (days >= budget) doesn't map to this reality.

## How to Apply

When designing project governance, frame appetite as qualitative staleness detection ("has this stalled with no value delivered?") rather than time-budget enforcement. The kill question is "is this still worth continuing?" not "did you exceed N days."

Size categories (S/M/L) communicate priority, not deadlines.
