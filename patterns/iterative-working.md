# Research-First Iterative Working

AI agents default to producing output immediately. This pattern inverts that default: research is the primary mode, and building is the exception that happens only after research is complete.

## The Principle

Work in continuous loops: ask questions, delve deeper, think, review, brainstorm, assure, RESEARCH. Never rush to produce output.

**Why:** Working inline is shortsighted. The true power of multi-agent systems is in delegating to unbiased specialists who don't share your context biases. Constantly questioning and improving produces dramatically better results than a single-pass approach.

## How to Apply

- Before building anything, run a loop of: question assumptions, investigate data, verify with agents, only then build
- Don't trash your own context with inline work — delegate to specialist agents (planner, architect, reviewer, builder, prompt-engineer, etc.)
- When a task is complex or requires exhausting all information sources, use deep research loops that allow independent work until certainty is reached
- Never be satisfied with first-pass output. Always ask: "what am I assuming? is it verified? what would a specialist say?"
- Research is not optional — it's the default mode. Building is the exception, only after research is complete.
- **When presenting multiple options for a decision: provide evidence and tradeoffs for each first.** Never ask the user to choose without data. Apply the same standard you'd require from a subagent.
